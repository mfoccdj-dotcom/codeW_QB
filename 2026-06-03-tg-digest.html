中国和美国之间的差距：其实不在顶层

不看过程只看结果，中国建国七十多年，就把世界重新拉回到了“两级结构”，这一点本身就已经说明问；

而中国的崛起，几乎是这个星球上无法被其他任何国家复刻的路径型奇迹，这只能说明一件事：中国的顶层是世界级的顶层。

中国和美国的之间的差距，更不在底层

前段时间小红书的“中美对账”已经很直观了：普通人的生活状态并没有本质差别；

中国有美团、滴滴、跑腿，美国有一天三份工；中国的牛马是牛马，美国的牛马同样是牛马;

甚至在大政府、大工业体系下，中国底层的生活成本和兜底能力，在很多维度上反而更友好。

中国和美国之间真正的差距，在中间层

美国有特斯拉、有英伟达，有马斯克、有黄仁勋，有一整套“允许你为了未来而赌”的土壤；

而中国的中间层企业家，太多精力花在了放贷、内卷、抄路径、抢存量上——最擅长的不是创造新蛋糕，而是精准地切老百姓的菜篮子；

所以诞生了，例如三大运营商、例如各种新能源车企、例如“免费中国”游戏这些让人恨的牙痒痒的傻比企业，你说中国人不聪明吗？不聪明怎么可能花样百出的去坑老百姓?

只是聪明才智更多被引导去做“确定性套利”，而不是“不确定性创造”。

但说到底，这件事本身并不反常，一个国家建国两百多年，富过，也一直在富；一个国家建国七十多年，长期处在贫穷里，最近几十年才刚刚摸到富裕的门槛；

如果把国家类比成一个人的。

一个穷了七十多年的人，很难一下子拥有那种从容、松弛、敢于试错的精神状态；

而一个已经富裕了两百多年的人，底气、耐心、冒险欲，本身就是被生活长期“训练”出来的。

穷人每天想的是温饱；富人早就不需要再为温饱焦虑，甚至连欲望本身都被满足过头了；

于是他们开始把精力投向更远的东西：探索未知、改变世界、留下点什么。

作为一个老中我想说的是：我们真正缺的不是方向，也不是能力，更不是聪明才智，只是时间——足够长、足够稳定、足够普遍的富裕时间、让我们能够更加从容一些的时间而已。

假如你使用Claude Code 配合本地模型，你会发现本地模型的推理速度下降大约90%，是因为 CC 自动在请求中添加了一个 Attribution Header，这个 header 会导致本地推理服务的 KV Cache 完全失效，从而使本地模型的推理速度下降 约 90%，一旦失效，相当于退化到几乎不使用缓存的状态，速度变得非常慢。

解决方案是：~/.claude/settings.json 在 env 字段中添加 "CLAUDE_CODE_ATTRIBUTION_HEADER": "0"

同时 Unsloth 这篇文章非常好 ：https://unsloth.ai/docs/basics/claude-code 教你怎样使用 Claude Code 运行本地模型，里面各种坑都说到了，非常翔实，很棒，如果有在本地运行模型的朋友，可以收藏看一看。

6、用 flomo MCP 搜索素材写稿

现在可以直接让它把相关笔记都捞出来，再按主、观点、案例、金句整理。

这对我这种长期写作的人很有用。

因为真正有价值的内容，不是临时编出来的。

而是过去长期观察、思考、实践之后，被重新组织出来的。

7、用飞书CLI存稿入库

写好的稿件，我会通过飞书 CLI 直接存入飞书知识库。

不用再手动复制、粘贴、改格式、建文件。这类动作单次看起来不大，但每天重复，非常消耗人。

Codex 最适合干重复、琐碎、低创造力，但又必须做对。能自动化的，就不要用自己的注意力去换。

8、建立内容创作系统日报

我用 Codex 设置了一个内容创作系统轻量日报。

每天自动帮我回顾，哪些选可以推进？哪些素材值得捞出来？哪些草稿需要继续处理？哪些文章可以改成视频？哪些内容可以导向产品？哪些旧内容可以重新分发？

这个东西特别适合自媒体人。

因为自媒体最大的内耗之一，就是每天都觉得自己要重新开始。

但其实你过去积累过很多东西。只是需要一个系统每天帮你捞起来。
